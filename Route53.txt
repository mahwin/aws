IP 주소에 domain 이름을 붙여서 쉽게 사용할 수 있게 하는게 DNS임.


DNS 동작 구조

도메인 구입 => 도메인과 IP 연결 => 도메인 IP를 클라이언트가 알아내는 것

1. 도메인 구입

ex) example.com을 등록대행자를 통해서 사야함.
.com .org => top 레벨 도메인이라고 함. 저것마다 관리하는 기관이 다름. 등록대행자가 저 탑 레벨 도메인을 관리한는 등록소에게 살려고 한다는 것을 알려주고 등록소에서 등록대행자에게 수수료를 받고 사용할 수 있는 권리를 주면 대행자가 다시 등록자에게 도메인을 양도함.

2. 구매한 도메인을 ip랑 연결하는 것!
도메인 네임 서버를 장만하고 (대신해주는 곳도 많음) 아이피와 연결
example.com의 주소는 93.184.216.34라고 설정하고 등록소에게 우리가 산 도메인ㅇ을 등록소에게 어디 도메인 네임 서버에 등록햇는지 알려줘여한느데 그것을 등록대행자가 실제로 대신 알려줌. 그러면 등록소는 자기의 도메인과 ip를 맵핑하고 그것의 도메인 네임 서버도 맵핑해놓음.

3. 클라이언트가 어떻게 example.com을 접근하려고 했을때 93.184.216.34라는 주소를 알 수 있느냐?.
DNS Server의 ip를 알고 있어서 거기로 들어가서 도메인에 해당하는 ip가 무엇인지 물어봄. (실제로는 훨씬 복잡함, 여러 과정을 거침)
